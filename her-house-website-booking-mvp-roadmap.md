# Her House Website & Booking MVP Roadmap

## 1. Project Summary

Her House is a luxury wellness club for women in Vera, Tbilisi.

The brand is not just a Pilates studio. It is a feminine, premium, social wellness space built around reformer Pilates, movement, connection, beauty, and belonging.

The main brand message is:

> **More than a workout. It's your place.**

The website must support three business goals:

1. Convert Instagram traffic into bookings.
2. Make Her House look premium, trustworthy, and emotionally desirable.
3. Build a strong local SEO foundation for searches like "Pilates Tbilisi," "Reformer Pilates Tbilisi," and "Pilates Vera Tbilisi."

The first version should be an MVP, but not a cheap or unfinished MVP. It should be small, elegant, fast, mobile-first, and conversion-focused.

## 5. Visual Brand Direction

### Overall Look

The website should feel soft, warm, feminine, premium, and editorial.

It should look closer to a boutique hotel, beauty studio, private women's club, or luxury wellness lounge than a standard gym website.

### Color Palette

Recommended MVP palette:

| Role | Hex | Usage |
|---|---:|---|
| Deep Burgundy | `#6A3038` | Primary buttons, footer, important headings, active states, premium accents |
| Wine Brown | `#4A2227` | Dark text, luxury sections, strong contrast moments |
| Dusty Rose | `#B77A82` | Soft accents, icons, highlights, hover states, secondary details |
| Blush Pink | `#E7C4C6` | Emotional background sections, soft cards, decorative elements |
| Warm Cream | `#F6EEE8` | Main website background |
| Nude Beige | `#D6BFB4` | Borders, dividers, pricing cards, gentle UI structure |
| Soft White | `#FFF9F5` | Clean content cards, booking forms, readable areas |
| Espresso | `#2D2321` | Primary body text |

### Color Usage Rule

Do not overuse bright pink.

The site should mainly use warm cream, soft white, burgundy, nude beige, dusty rose, and espresso. Pink should be emotional, not childish.

## 10. Class Schedule Page

### Schedule Page Goal

The schedule page must make booking extremely easy.

The page should show only relevant classes and should not overwhelm the visitor.

### Booking Window Rules

The owner wants:

- Minimum booking time: 24 hours in advance.
- Maximum booking time: 7 days in advance.

So the schedule page should:

- Show classes for the next 7 days.
- Allow booking only if the class starts more than 24 hours from now.
- Show "Booking closed" for classes within 24 hours.
- Hide or disable classes outside the 7-day window.
- Show available spaces for each class.

### Class Card Information

Each class card should include:

- Class name
- Day and date
- Time
- Duration
- Instructor, if available
- Available spaces
- Class level, if available
- Book button

Example:

> **Reformer Flow**  
> Monday, 18:00 - 50 min  
> All levels - 3 spaces left  
> Instructor: Anna  
> **Book Class**

### Class Card States

#### Open

Show:

> **3 spaces left**  
> Button: **Book Class**

#### Full

Show:

> **Class full**  
> Button: **Join Waitlist** if waitlist exists, otherwise disabled.

#### Less Than 24 Hours Away

Show:

> **Booking closed**  
> Button disabled.

#### Cancelled

Show:

> **Cancelled**  
> Button disabled.

#### Not Yet Open

If showing future classes beyond 7 days later:

> **Bookings open 7 days before class**

For MVP, it may be simpler to not show classes more than 7 days away.

## 11. Booking Flow MVP

### Main Booking Flow

Instagram ad  
-> Website landing page  
-> Class Schedule  
-> Book Class  
-> Enter name and WhatsApp number  
-> Confirmation screen

### Booking Form Fields

Required:

- Full name
- WhatsApp number

Optional:

- First time at Her House? Yes/No
- Notes for instructor
- Injury/pregnancy note

### Booking Validation

The system must check:

1. Is the class within the next 7 days?
2. Is the class more than 24 hours away?
3. Are spaces still available?
4. Has the same WhatsApp number already booked this class?
5. Is the WhatsApp number valid enough to contact?
6. Is the class still active/open?

After successful booking:

> **You're booked <3**

Show:

- Class name
- Date
- Time

### Confirmation Message Style

The confirmation should feel branded, not technical.

Do not say:

> "Submission successful."

Say:

> **You're booked <3**

## 12. Google Calendar Role

The owner suggested Google Calendar as the easiest way to track the schedule.

This is reasonable for MVP, but Google Calendar alone should not be the full booking system if the site must show spaces left and prevent overbooking.

### Recommended Approach

Use the website/admin system as the source of booking logic.

Use Google Calendar as a supporting view/sync tool.

### Why

The website needs to control:

- Capacity
- Spaces left
- 24-hour rule
- 7-day rule
- Duplicate bookings
- Booking form data
- Waitlist later
- Admin view

Google Calendar can help with:

- Owner visibility
- Schedule overview
- Personal planning
- Class reminders
- Optional sync

### MVP Recommendation

For first version:

- Build a simple internal class schedule/admin database.
- Add optional Google Calendar sync later if it increases owner convenience.
- Do not block MVP launch on complex calendar sync.
